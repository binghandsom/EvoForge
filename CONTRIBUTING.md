# Contributing to EvoForge

Thanks for your interest in contributing to EvoForge. We welcome engineers, researchers, and builders who want to help push self-evolving AI systems forward.

## Ways to Contribute
- Open issues to report bugs, propose ideas, or discuss design decisions.
- Submit pull requests for features, fixes, tests, or documentation improvements.
- Share real-world use cases, benchmarks, and feedback.

## Before You Start
- Search existing issues to avoid duplicates.
- For large changes, open an issue first so we can align on direction.

## Pull Request Process
- Fork the repository and create a feature branch.
- Keep PRs focused and small when possible.
- Add or update tests where relevant, and keep existing tests green.
- Update documentation when behavior or APIs change.
- Link related issues or discussions in the PR description.

## Development Notes
- Follow existing coding style and project conventions.
- If you add dependencies, explain the rationale and document any setup changes.
- If you make a breaking change, call it out clearly in your PR.

## Community
Please follow the guidelines in [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
